# 带声调精确匹配与不限声调兜底 FST 设计

## 1. 背景

当前项目使用 `lexicon.txt` 对 ASR 输出的中文进行最大前向分词，并将文字转换为带声调拼音；随后使用 `replace.fst` 将命中的拼音规则替换为目标文字。

现有 FST 规则只支持带声调拼音的精确匹配。例如目标规则为 `luan3chao2=卵巢` 时，输入 `luan4chao2` 无法命中。

本次业务词汇来自 `超声词汇.xlsx`。本设计使用 `lexicon.txt` 为 Excel 中的目标词汇生成带声调拼音，在保留带声调精确匹配的基础上增加不限声调匹配能力，同时避免在纯拼音存在多个目标时自动判断目标。

## 2. 目标

- 保留带声调拼音的精确匹配，且精确规则优先。
- 当声母、韵母相同，仅声调不同时，允许匹配唯一目标。
- 纯拼音对应多个不同目标时，禁止生成不限声调规则。
- 以 `超声词汇.xlsx` 作为本次长期词汇来源。
- 使用运行时相同的 `lexicon.txt` 分词和拼音规则生成新的 `mapping.txt`。
- 从审核后的新 `mapping.txt` 生成单个 `replace.fst`。
- 本地脚本与 README 中的 Colab 使用同一套生成逻辑。
- 临时运行时规则保持高于 FST 的优先级，并使用相同的精确/宽松策略。

## 3. 非目标

- 不支持 `z/zh`、`in/ing`、`an/ang` 等近音模糊匹配。
- 不处理 `NT`、`CDFI`、`S1段` 等中英混合规则。
- 不修改 `lexicon.txt` 的拼音格式。
- 不使用项目参与者提供的现有 `mapping.txt` 作为本次规则来源。
- 不穷举所有声调组合生成数百万行文本规则。
- 不让系统在多个目标之间根据上下文作判断。

## 4. 核心原则

### 4.1 lexicon 保持带声调输出

`lexicon.txt` 继续承担分词和文字转拼音职责，例如：

```text
乱潮 → luan4chao2
卵巢 → luan3chao2
```

为了支持不限声调匹配，不需要修改 `lexicon.txt`。

### 4.2 Excel 生成新的 mapping.txt

工具从 `超声词汇.xlsx` 提取、清洗并去重词汇，再使用 `lexicon.txt` 按项目运行时相同的最大前向分词规则生成带声调拼音。

例如 Excel 中存在：

```text
卵巢
肺静脉
腓静脉
```

生成新的 `mapping.txt`：

```text
luan3chao2=卵巢
fei4jing4mai4=肺静脉
fei2jing4mai4=腓静脉
```

不需要为错误声调人工增加：

```text
luan4chao2=卵巢
```

现有项目参与者提供的 `mapping.txt` 不参与本次生成。新 `mapping.txt` 经人工检查后，生成工具根据正确规则自动构建不限声调规则。

正式部署只加载生成后的 `replace.fst`，不再通过 `--rules-file mapping.txt` 重复加载同一批规则。

### 4.3 多音字修正

自动生成的拼音必须经过人工检查。如果 `lexicon.txt` 在专业语境中给出错误读音，应在 `lexicon.txt` 中增加或修正对应专业词条，然后重新生成 `mapping.txt`。

例如：

```text
错误：长径 → zhang3jing4
正确：长径 → chang2jing4
```

应在 lexicon 中维护正确的专业词读音，避免直接修改生成结果后造成下一次生成丢失修正，也保证运行时输入拼音与 FST 规则一致。

### 4.4 单个 FST、一次匹配

每条可兜底规则在同一个 FST 中包含两种路径：

```text
精确路径：luan3chao2 → 卵巢，优先级高
宽松路径：luan + 任意声调 + chao + 任意声调 → 卵巢，优先级低
```

运行时仍然输入带声调拼音，只调用一次 FST。精确路径和宽松路径均能匹配时，选择精确路径。

## 5. 无声调冲突规则

生成工具删除声调数字后进行分组。

### 5.1 唯一目标

```text
luan3chao2=卵巢
```

去掉声调后为 `luanchao`，且只有一个目标，因此生成：

- `luan3chao2 → 卵巢` 精确规则
- 任意声调的 `luan + chao → 卵巢` 宽松规则

### 5.2 多个目标

```text
fei4jing4mai4=肺静脉
fei2jing4mai4=腓静脉
```

去掉声调后均为 `feijingmai`，对应两个目标。因此：

- 保留两条带声调精确规则
- 不生成 `feijingmai` 宽松规则
- 在构建报告中记录冲突

系统只负责匹配，不负责在多个目标之间作判断。

## 6. FST 生成流程

生成工具放在项目内，Colab 只负责安装依赖、调用项目脚本和下载结果。

流程分为词汇准备和 FST 编译两段：

```text
超声词汇.xlsx + lexicon.txt
→ 提取纯中文词汇
→ 清洗并去重
→ 按运行时相同规则分词和生成带调拼音
→ 生成新的 mapping.txt
→ 人工检查多音字和专业读音
→ 必要时修正 lexicon.txt 并重新生成

审核后的 mapping.txt
→ 读取和格式校验
→ 生成所有带调精确规则
→ 按去调拼音分组
→ 为唯一目标组生成宽松规则
→ 合并并优化规则
→ 生成 replace.fst
→ 生成 fst-build-report.txt
```

生成依赖：

- Python
- Pynini
- OpenFST（由 Pynini 提供）
- 项目内的统一生成脚本
- `超声词汇.xlsx`
- `lexicon.txt`

词汇准备阶段必须读取 `lexicon.txt`，并复用当前项目的最大前向分词和单字回退规则。FST 编译阶段只读取审核后的新 `mapping.txt`。

## 7. mapping.txt 格式与校验

新 `mapping.txt` 由 Excel 和 lexicon 自动生成，格式为：

```text
带声调拼音=目标文字
```

生成工具和 FST 编译工具支持空行及以 `#` 开头的注释：

```text
# 妇科超声词汇
luan3chao2=卵巢
```

生成前执行以下检查：

1. 拼音键不能为空。
2. 目标文字不能为空。
3. 每个拼音音节必须以 `1` 至 `4` 结尾。
4. 完全相同的规则自动去重。
5. 相同带调拼音对应不同目标时，构建失败。
6. 纯拼音对应不同目标时，不构建宽松规则，但不影响精确规则生成。

词汇准备阶段还需要检查：

1. Excel 中的空单元格和重复词汇。
2. 词汇是否为本阶段支持的纯中文内容。
3. 每个中文字是否存在于单字 lexicon 中。
4. 生成的拼音音节数是否与中文字数一致。
5. 需要人工检查的多音字和专业词读音。

错误示例：

```text
luan3chao2=卵巢
luan3chao2=乱潮
```

生成工具必须报告具体行号并停止，不得自行选择目标。

## 8. 构建安全与报告

生成工具先写入临时输出，全部成功后再替换正式 `replace.fst`。任何错误都不得覆盖当前可用的 FST。

`fst-build-report.txt` 至少包含：

- Excel 非空词条数量
- 去重词条数量
- 本阶段纳入的纯中文词条数量
- 排除的中英混合词条数量及明细
- lexicon 缺失字和拼音生成异常
- 原始规则数量
- 有效规则数量
- 去重规则数量
- 精确规则数量
- 宽松规则数量
- 纯拼音冲突数量及明细
- 格式错误及对应行号
- FST 文件大小
- 构建耗时

## 9. 运行流程

```text
ASR 中文文本
→ lexicon 最大前向分词
→ lexicon 输出带声调拼音
→ 调用一次 replace.fst
→ 精确规则优先、宽松规则其次
→ 输出替换后的文字
```

例如：

```text
输入：左侧乱潮
拼音：zuo3ce4luan4chao2
规则源：luan3chao2=卵巢
结果：左侧卵巢
```

FST 只替换命中的拼音片段，前后的“左侧”等内容保持原样。没有任何规则命中的内容也保持原样。

## 10. 临时运行时规则

`--add-rule`、`--del-rule` 和 `--rules-file` 继续用于临时热修，优先级高于 `replace.fst`。

临时规则需要建立：

- 带声调精确规则集合
- 纯拼音唯一目标规则集合

执行原则与 FST 一致：

- 精确匹配优先
- 纯拼音只有一个目标时才允许宽松匹配
- 纯拼音对应多个目标时禁止宽松匹配

正式部署不应同时加载用于生成 FST 的完整 `mapping.txt`，避免重复处理。

## 11. 测试设计

### 11.1 精确匹配

```text
规则：luan3chao2=卵巢
输入拼音：luan3chao2
预期：卵巢，使用精确规则
```

### 11.2 不同声调匹配

```text
规则：luan3chao2=卵巢
输入拼音：luan4chao2
预期：卵巢，使用宽松规则
```

### 11.3 纯拼音冲突

```text
规则：
fei4jing4mai4=肺静脉
fei2jing4mai4=腓静脉

验证：
fei4jing4mai4 → 肺静脉
fei2jing4mai4 → 腓静脉
fei3jing4mai4 → 不进行宽松替换，保留原文
```

### 11.4 未定义规则

没有精确规则或宽松规则时，原文字保持不变。

### 11.5 构建工具

覆盖以下情况：

- 正常生成
- 重复规则去重
- 相同带调拼音对应不同目标时失败
- 纯拼音冲突时仅生成精确规则
- 错误格式显示行号
- 构建失败不覆盖旧 FST
- 本地与 Colab 生成结果功能一致
- Excel 词汇提取、清洗和去重正确
- 词汇拼音生成行为与 C++ 运行时一致
- lexicon 缺字时报告具体词汇和字符
- 专业多音字修正后能够稳定重新生成

## 12. 性能验收

使用同一份 `mapping.txt` 分别生成：

- 仅包含带调精确规则的基准 FST
- 包含精确规则和宽松规则的目标 FST

比较指标：

- FST 文件大小
- 构建耗时
- 加载耗时
- 单句匹配耗时
- 长文本吞吐量

验收目标：增加宽松规则后，匹配速度相对精确规则基准下降不超过 20%。若超过该范围，先优化规则生成方式，再进入正式使用。
